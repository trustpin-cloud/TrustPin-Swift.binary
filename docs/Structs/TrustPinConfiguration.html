TrustPinConfiguration

public struct TrustPinConfiguration : Sendable

Encapsulates all configuration options for a TrustPin instance.

Pass a TrustPinConfiguration to setup(_:) (or the static setup(_:autoRegisterURLProtocol:) for the default instance) instead of individual parameters. This keeps call sites stable as new options are added — new properties always carry default values so existing code never needs to change.

Required credentials

The three credentials (organizationId, projectId, publicKey) are set once in the initializer and are immutable. They identify the pinning payload on the TrustPin CDN and authenticate its signature.

Optional behaviour

All other settings are var with sensible defaults and can be changed before passing the configuration to setup:

var config = TrustPinConfiguration(
    organizationId: "your-org-id",
    projectId: "your-project-id",
    publicKey: "your-base64-public-key"
)
config.mode = .permissive
config.logLevel = .debug

try await TrustPin.setup(config)

Custom configuration URL

For self-hosted or BYOK deployments, supply a configurationURL pointing to a signed JWS payload. CDN-managed projects should leave this nil.

var config = TrustPinConfiguration(
    organizationId: "org",
    projectId: "proj",
    publicKey: key
)
config.configurationURL = URL(string: "https://your-server.com/pins.jws")

  • mode
